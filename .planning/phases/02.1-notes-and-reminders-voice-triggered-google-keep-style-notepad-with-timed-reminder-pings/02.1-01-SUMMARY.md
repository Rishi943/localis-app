## Decisions Made
- Used plain `CREATE INDEX IF NOT EXISTS idx_notes_due_active ON notes(due_at, dismissed)` without a WHERE clause — the plan noted SQLite partial index may fail on older versions; plain index avoids that risk with negligible overhead given the expected note count
- Hard delete via `DELETE FROM notes WHERE id = ?` rather than soft-delete — notes are short-lived; reminder lifecycle uses `dismissed=1` which is already distinct behavior
- All color and type values validated at POST/PATCH time with explicit 400 responses to keep DB data clean for the frontend palette rendering in plan 02.1-03

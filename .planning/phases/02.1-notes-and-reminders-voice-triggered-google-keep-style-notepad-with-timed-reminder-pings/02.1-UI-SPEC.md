# Phase 02.1 — UI Design Contract

> Visual and interaction contract for Notes and Reminders notepad. Generated by gsd-ui-researcher from CONTEXT.md decisions, verified against existing Midnight Glass design system.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | none (Midnight Glass custom CSS) |
| Icon library | Inline SVG symbols (existing app pattern) |
| Font | Inter (400/500/600) + JetBrains Mono |

**Design identity:** Midnight Glass — pure black foundation with liquid glass translucency (backdrop-filter: blur 24px, saturate 180%). All new components use canonical CSS variables from `UIUX/DESIGN.md`.

## Spacing Scale

Declared values (8-point scale, all multiples of 4):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, inline note text padding |
| sm | 8px | Card inner padding, compact spacing |
| md | 16px | Default card padding, grid gaps |
| lg | 24px | Section padding within notes dashboard |
| xl | 32px | Layout gaps between grid columns |
| 2xl | 48px | Major section breaks (header/footer) |
| 3xl | 64px | Page-level spacing (notes dashboard margin) |

Exceptions: None — all notes UI uses 8-point scale exclusively.

## Typography

| Role | Size | Weight | Line Height | Usage |
|------|------|--------|-------------|-------|
| Body (note content) | 14px | 400 | 1.5 | Note card text content |
| Label (card header/time) | 12px | 500 | 1.4 | Card metadata, due date display |
| Heading (section header) | 16px | 600 | 1.3 | Dashboard section titles (if any) |
| Card title (inline edit) | 14px | 500 | 1.4 | Note preview in create/edit mode |

**Rationale:** Inherits from app body baseline (14px). Labels and timestamps use 12px to signal secondary importance. No display-scale text in notes UI.

## Color

| Role | Value | Usage |
|------|-------|-------|
| Dominant surface (60%) | `rgba(0, 0, 0, 1.0)` | Notes dashboard background (pure black) |
| Secondary surface (30%) | `var(--bg-panel): rgba(15, 15, 15, 0.45)` | Note card backgrounds |
| Accent primary (10%) | `#5A8CFF` | Card color tint overlays, focus states |
| Accent secondary (reminder overdue) | `#F59E0B` | Amber tint for overdue card badge + border |
| Text primary | `var(--text-primary): #ffffff` | Note content, card text |
| Text secondary | `var(--text-secondary): #a1a1aa` | Timestamps, metadata labels |
| Destructive | Not used in v1 (delete deferred) | — |

**Accent reserved for:**
- Card color tint options (6 translucent dark tints: default near-black, deep blue, dark teal, amber night, rose night, mauve glass)
- Focus ring on card edit mode
- Overdue reminder badge glow/pulse animation

**Card color tints** (6 options, all dark translucent, tuned to Midnight Glass):
1. **Default** — near-black glass (`rgba(15, 15, 15, 0.5)`)
2. **Deep Blue** — `rgba(30, 58, 138, 0.35)` with blue border tint
3. **Dark Teal** — `rgba(20, 84, 92, 0.35)` with teal border tint
4. **Amber Night** — `rgba(120, 53, 15, 0.35)` with amber border tint (also used for overdue state)
5. **Rose Night** — `rgba(108, 28, 55, 0.35)` with rose border tint
6. **Mauve Glass** — `rgba(76, 29, 149, 0.35)` with mauve border tint

All card backgrounds use glass recipe: `backdrop-filter: var(--glass-filter)` + `border: 1px solid rgba(255, 255, 255, 0.15)`.

## Component Inventory

### Notes Panel Container
- Full-screen overlay replacing main content area
- Grid layout: 2-4 columns (responsive, based on viewport width)
- CSS columns or masonry JS library for waterfall effect
- Header: "Notes" title + optional search (deferred v2) + grid/list toggle (deferred v2)
- Footer: None required (notes infinite-scroll in grid)
- Scrollbar: Ghost scrollbar recipe (`.04`/`.08` opacity, 3px wide, transparent track)

### Note Card
- Default size: ~280px width × auto height (masonry auto-scales)
- Border radius: 12px
- Padding: `md (16px)` all sides
- Background: Glass recipe + color tint overlay (6 options)
- Hover state: Slightly increased opacity (e.g., `0.5` → `0.6`), trash icon visible
- Click-to-edit: Card enters edit mode with inline textarea, save on blur
- Edit mode border: Focus ring using accent color (`#5A8CFF`)
- Timestamp: `label (12px)` gray text, always visible at card bottom
- Trash icon: 16×16 SVG, opacity `0.5` by default, `1.0` on hover, positioned top-right corner with `8px` offset

### Reminder Badge (on reminder cards)
- Clock emoji or SVG symbol (⏰) at card bottom-left
- Badge style: Small dark box with clock + formatted time (e.g., "⏰ Tomorrow 9:00 AM")
- Overdue state: Amber tint (`#F59E0B`), pulsing animation (opacity 0.7 → 1.0 over 1.5s infinite)
- Font: `label (12px)`, text color inherits secondary

### Create Card (empty dashed card)
- Dashed border: `2px dashed rgba(255, 255, 255, 0.2)`
- Centered text: "Add a note..." in secondary color
- Click to enter inline creation mode
- Edit mode: Textarea auto-focused, placeholder "Add a note..."
- Save on blur or Enter key

### Voice Confirmation Message (in main chat)
- Appears in chat history as assistant message
- Type: `"status"` or `"assistant"` (executor to determine)
- Content: "Got it. Note saved. [8-char note preview]..." OR "Reminder set for [time]. [preview]"
- No action chips (plain message)

### Reminder Notification (OS-level)
- Uses browser Web Notification API
- Title: "Reminder: [note content, first 60 chars]"
- Body: "[Formatted time] — [app name]"
- Icon: App favicon or custom reminder icon (executor discretion)
- Click: Brings app to foreground, scrolls to notes panel

### Reminder Poll Badge (on Notes icon in RSB)
- Badge element: Small circle with count number
- Visible only if count > 0
- Color: Amber (`#F59E0B`)
- Position: Top-right corner of Notes icon
- Font: `label (12px)` white text, bold

### Overdue Card Glow
- CSS animation: `pulse` (opacity 0.7 → 1.0 over 1.5s infinite)
- Applied to overdue reminder cards with amber card tint
- Box-shadow: `0 0 12px rgba(245, 158, 11, 0.4)` (amber glow)

## Interaction Patterns

### Create Note (Voice)
1. User says: "Hey Jarvis, add note: [content]"
2. Router LLM parses intent, returns `{type: "note", content: "...", due_at: null}`
3. Backend saves to `notes` table
4. TTS: "Got it. Note saved."
5. Chat message appears with confirmation + 8-char preview
6. Frontend calls `notesUI.refresh()` to reload grid
7. New card appears in grid (position: varies per masonry layout)

### Create Reminder (Voice)
1. User says: "Hey Jarvis, remind me to [action] [time]"
2. Router LLM parses intent + resolves time relative to server timezone, returns `{type: "reminder", content: "...", due_at: "ISO8601"}`
3. Backend saves to `notes` table with `note_type='reminder'`
4. TTS: "Reminder set for [formatted due_at]."
5. Chat message appears with confirmation + preview
6. Frontend calls `notesUI.refresh()` to reload grid
7. New reminder card appears with clock badge

### Reminder Delivery (Frontend Poll)
1. Frontend polls `GET /notes/due` every 30 seconds
2. Backend returns fired reminders: `[{id, content, due_at, dismissed}]` where `due_at <= now` AND `dismissed = 0`
3. For each new fired reminder:
   - Play chime sound (executor picks audio file, ~0.5s, subtle system tone)
   - Request browser Notification permission if not granted
   - Fire Web Notification API with title + body
   - Refresh notes grid to show overdue badge on reminder card
   - Add badge count to Notes icon in RSB
4. User clicks reminder card or Web Notification notification:
   - Card scrolls into view (if grid scrolled)
   - Card glow/pulse animation triggers
5. User can manually dismiss by clicking overdue card or via delete (v2 — v1 delete deferred)

### Edit Note (Inline)
1. User clicks on existing card
2. Card content enters edit mode: textarea with current note text
3. Focus ring: accent blue (`#5A8CFF`)
4. Save triggers on blur or Ctrl+Enter
5. Backend: `PATCH /notes/{id}` with updated content
6. Card reverts to display mode, updated text shown
7. Error: Toast "Failed to save note" with retry option

### Delete Note (Hover)
1. User hovers over card: trash icon fades in (opacity 0.5 → 1.0)
2. Click trash icon: immediate delete (no confirmation modal in v1)
3. Backend: `DELETE /notes/{id}`
4. Frontend: Card fades out, grid reflows
5. Toast (5s): "Note deleted. [Undo] [Dismiss]" (Undo deferred to v2, dismiss closes toast)
6. After 5s timeout: permanent delete (no actual undo in v1)

### Color Selection (Click card header/color indicator — deferred v2)
Deferred to v2. v1 has hardcoded color assignment or no color picker UI.

### Pinned Cards (Pin icon hover — deferred v2)
Deferred to v2. v1 pins recorded in schema but no UI to toggle.

## Responsive Behavior

**Desktop (≥1080px):** 3-4 columns, grid gap `lg (24px)`
**Tablet (768px–1079px):** 2-3 columns, grid gap `md (16px)`
**Mobile (<768px):** 1 column, card width 100% with `md (16px)` padding from edges

All layouts preserve masonry reflow behavior — cards maintain their original creation order but fill space efficiently.

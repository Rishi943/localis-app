Action chips (from RESEARCH.md Code Examples):
```javascript
function addMessageActionChips(msgRowEl, plainText) {
    const chips = document.createElement('div');
    chips.className = 'msg-actions';
    chips.innerHTML = `
        <button class="msg-action-chip" data-action="copy">Copy</button>
        <button class="msg-action-chip" data-action="regen">Regenerate</button>
        <button class="msg-action-chip" data-action="continue">Continue</button>
    `;
    chips.querySelector('[data-action="copy"]').onclick = () =>
        navigator.clipboard.writeText(plainText);
    chips.querySelector('[data-action="regen"]').onclick = () =>
        api.chat(state.lastUserMessage);
    chips.querySelector('[data-action="continue"]').onclick = () =>
        api.chat('Please continue.');
    msgRowEl.appendChild(chips);
}
```

**4. Thinking block collapse** — Find `parseThinking()` in app.js. Update it to handle both `<thinking>` and `<think>` tag formats:
```javascript
function parseThinking(text) {
    return text.replace(/<think(?:ing)?>([\s\S]*?)<\/think(?:ing)?>/gi, (_, content) => {
        const id = 'think-' + Math.random().toString(36).slice(2, 8);
        return `<div class="thinking-pill" onclick="document.getElementById('${id}').classList.toggle('expanded')">▸ Thinking...</div><div class="thinking-body" id="${id}">${content.trim()}</div>`;
    });
}
```
This ensures: content is hidden by default (`.thinking-body` has `display:none`), clicking the pill toggles `.expanded` which sets `display:block`. The content never shifts up after the think phase ends because the pill always takes up its own line.

Call `parseThinking()` as the FIRST step in the post-stream markdown processing (before `marked.parse()`), passing the full raw text — so thinking blocks are extracted before markdown renders them.

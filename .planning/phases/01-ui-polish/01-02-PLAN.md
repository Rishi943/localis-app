Bug 1 — Session list CSS class mismatch:
- app.js creates elements: `div.className = 'session-item ...'`
- app.css defines: `.sess-item` (NOT `.session-item`)
- Fix: change 'session-item' → 'sess-item' in className assignment; also change delete button class if it exists

Bug 2 — Right sidebar collapse: two conflicting systems:
- CSS uses `.rsb.collapsed` (width-based) — this is the CORRECT system
- JS toggleSettings() uses `.visible` class — NO CSS rule for `.rsb.visible` exists
- Fix: toggleSettings(show) should call `rsb.classList.toggle('collapsed', !show)` instead of toggling `.visible`

Bug 3 — Auto-scroll targeting wrong element:
- `els.chatHistory` → `#chat-history` = the INNER `.chat-inner` div (no overflow, cannot scroll)
- The scrollable viewport is `#chat-zone` (has `overflow-y: auto`)
- Fix: scroll operations must target `document.getElementById('chat-zone')` or a cached `els.chatZone`

Correct scrollToBottom implementation:
```javascript
const scrollToBottom = () => {
    const chatViewport = document.getElementById('chat-zone');
    if (!chatViewport) return;
    const isNearBottom = chatViewport.scrollHeight - chatViewport.scrollTop - chatViewport.clientHeight < 120;
    if (isNearBottom) chatViewport.scrollTop = chatViewport.scrollHeight;
};
```

**Fix 1: toggleSettings() sidebar collapse**

Find the `toggleSettings` function. The research shows it currently toggles a `.visible` class on the right sidebar — a class that has no CSS rule. Replace the body of toggleSettings to use the CSS `.collapsed` approach instead:

```javascript
const toggleSettings = (show) => {
    // show=true → open RSB; show=false → close RSB
    const rsb = els.rightSidebar;
    if (!rsb) return;
    const shouldOpen = (show !== undefined) ? show : rsb.classList.contains('collapsed');
    rsb.classList.toggle('collapsed', !shouldOpen);
    if (els.state) state.rightSidebarOpen = shouldOpen;
};
```

Also find the `toggleRight()` function (wired to `#right-sidebar-toggle`, the 20px strip). Verify it also uses `classList.toggle('collapsed')` — if it uses `.visible` or `.hidden` fix it to use `.collapsed` as well.

After these changes, verify that nothing in app.js sets `.rsb.visible` or `.right-sidebar.visible` — search for `.visible` being added to the right sidebar element and remove those calls.

**Fix 2: Auto-scroll to correct viewport element**

Find the `scrollToBottom` function (or equivalent inline scroll logic used during streaming). The current bug: it scrolls `els.chatHistory` which is `#chat-history` (the inner non-scrollable container).

```javascript
    if (atBottom) vp.scrollTop = vp.scrollHeight;
}
```

Replace ALL usages of `els.chatHistory.scrollTop = ...` or `els.chatHistory.scrollHeight` that are meant to scroll the chat view with calls to this corrected `scrollToBottom()` function or the direct `els.chatZone` target.
